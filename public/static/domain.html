Research Domain

The Problem Space, the Gap, and our Approach

From literature to methodology — how this research was conceived, designed and validated.

Literature Survey

Existing modernization approaches — code translation, API wrapping, screen scraping and full re-engineering — all assume access to source code, structured APIs or stable UI elements. None address the constraints of closed-binary, vendor-abandoned legacy systems still running mission-critical operations in healthcare, banking and government. Recent advances in LLMs, vision-language models, and computer-use agents (e.g. Claude/GPT-4V) suggest a new path: treat the legacy app as a black-box environment and orchestrate it from the outside.

Research Gap

There is no end-to-end framework that combines (1) zero-intrusion observation of legacy desktop apps, (2) self-optimising agentic planning, (3) deterministic and recoverable execution, and (4) modern LLM-generated frontends — into a single deployable middleware. Existing RPA tools (UiPath, Power Automate) are brittle and not learning-based; existing AI agents lack stateful long-horizon orchestration on Windows desktop targets.

Research Problem

How can a modern, AI-native interface be safely and reliably layered on top of a legacy Windows desktop application (e.g. a Visual Basic 6 hospital management system on Windows XP) without modifying its source code, while guaranteeing task accuracy, recoverability and adaptability?

Research Objectives

1. Design a four-layer agentic middleware (LLM frontend, planner, orchestrator, RPA bots).  2. Implement SOACRS — a self-optimising contextual-bandit-based agent router.  3. Build a stateful DAG execution engine with rollback and observability.  4. Deliver a CV+OCR bot layer that operates legacy GUIs reliably.  5. Empirically evaluate the framework against a real VB6 hospital system across 150+ task executions.

Methodology

Design Science Research methodology: requirements were elicited from real legacy hospital workflows; the four-layer architecture was iteratively designed and prototyped; each layer was evaluated independently (LLM accuracy, planner regret, orchestrator reliability, bot success rate); finally, end-to-end evaluation was performed across 150 task executions, achieving a 97% success rate. Failure modes are categorised and fed back into the SOACRS reward signal.
